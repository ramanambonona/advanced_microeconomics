Introduction

Le Advanced Microeconomics Solver (AMS) est un solveur symbolique généralisé pour l'optimisation microéconomique d'ordre n et temps t. Il permet de résoudre des problèmes d'optimisation statique, dynamique et stochastique en utilisant des méthodes symboliques avancées.

Fonctionnalités principales

  • Généralisation d'ordre n : Support pour n biens et paramètres
  • Optimisation temporelle : Problèmes dynamiques d'ordre t
  • Calcul symbolique : Solutions avec SymPy
  • Trois modules : Statique, Dynamique, Stochastique
  • Visualisations : Graphiques interactifs avec Matplotlib
  • Explications détaillées : Étapes de résolution pédagogiques

Note : AMS est particulièrement adapté pour l'enseignement et la recherche en économie, permettant d'illustrer les concepts théoriques avec des solutions précises.

Module d'Optimisation Statique

Le module statique permet de résoudre des problèmes d'optimisation microéconomique classiques avec n biens.

Fonctions d'utilité supportées

Cobb-Douglas

$$ U = A \prod_{i=1}^n x_i^{\alpha_i} $$

Leontief

$$ U = \min(\frac{x_1}{a_1}, \dots, \frac{x_n}{a_n})$$

CES

$$ U = \left(\sum _{i=1}^n \alpha_i x_i^{\rho}\right)^{\frac{1}{\rho}}$$

CRRA

$$ U = \sum_{i=1} ^n \frac {x_i^{1-\gamma}-1}{1-\gamma}$$

Problèmes résolus

  • Demandes marshalliennes : Maximisation d'utilité sous contrainte budgétaire
  • Demandes hicksiennes : Minimisation des dépenses pour un niveau d'utilité donné
  • Élasticités : Élasticités-prix, revenu, croisées
  • TMS : Taux Marginal de Substitution
  • Équation de Slutsky : Décomposition effet substitution/effet revenu

Exemple : Pour résoudre un problème Cobb-Douglas avec 3 biens, spécifiez n=3 et les paramètres α₁, α₂, α₃. Le solveur générera automatiquement les symboles x₁, x₂, x₃, p₁, p₂, p₃.

Module d'Optimisation Dynamique

Le module dynamique résout des problèmes d'optimisation intertemporelle en temps discret et continu.

Exemple de problème

$$ \begin{align*} \max \quad & \sum_{t=0}^{\infty} \beta^t \log(c_t) \\ \text{s.c. } \quad & k_{t+1} = A k_t^{\alpha} + (1-\delta)k_t - c_t \end{align*} $$

Le solveur génère les équations d'Euler et trouve la solution symbolique.

Module d'Optimisation Stochastique

Le module stochastique traite des problèmes d'optimisation sous incertitude, notamment avec des processus Markoviens.

Méthodes implémentées

  • Processus Markoviens discrets
  • Équations d'Euler stochastiques
  • Simulation Monte Carlo
  • Distribution stationnaire
  • Programmation dynamique stochastique

Configuration de la matrice de transition

La matrice de transition P doit être stochastique : chaque ligne doit sommer à 1.

Exemple pour 2 états :

P = [[0.9, 0.1],  # De l'état 1 vers états 1 et 2
     [0.2, 0.8]]  # De l'état 2 vers états 1 et 2

Référence API

L'API REST permet d'intégrer le solveur dans d'autres applications.

Endpoints principaux

POST /solve/static

Résout un problème d'optimisation statique

{
  "problem_type": "marshallian",
  "utility_type": "cobb-douglas",
  "n_goods": 3,
  "parameters": {"alpha1": 0.3, "alpha2": 0.3, "alpha3": 0.4},
  "constraints": ["p1*x1 + p2*x2 + p3*x3 - R"]
}
POST /solve/dynamic

Résout un problème d'optimisation dynamique

POST /solve/stochastic

Résout un problème d'optimisation stochastique
